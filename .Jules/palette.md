## 2024-05-17 - Accessible Custom Toggle Switches
**Learning:** When implementing custom toggle switches (like the theme toggler in `index.html`), using `display: none;` on the hidden native checkbox input breaks accessibility completely. Screen readers skip elements with `display: none;`, meaning users cannot interact with or understand the toggle. Furthermore, simply hiding the input means it cannot receive focus, thus preventing keyboard navigation.
**Action:** Always use the 'visually hidden' CSS pattern (`position: absolute; opacity: 0; width: 0; height: 0;`) instead of `display: none;` for visually hidden form inputs. Additionally, always explicitly provide focus styles for the custom UI elements when the hidden input receives focus (e.g., using `:focus-visible` to add an outline to the adjacent `.slider` element) to ensure keyboard users can navigate to and operate the control.
