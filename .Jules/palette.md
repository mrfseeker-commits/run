## 2024-05-24 - Accessible Custom Toggle Switches
**Learning:** `display: none` completely removes an input from the accessibility tree, making custom toggle switches unreachable for keyboard and screen reader users. In Playwright scripts, `page.wait_for_selector` fails on visually hidden elements (e.g., `opacity: 0`), so you have to target adjacent visible elements or use `wait_for_timeout`. When verifying `:focus-visible` accessibility states, actual keyboard event simulation (e.g., `page.keyboard.press("Tab")`) must be used instead of programmatic `.focus()` to trigger the visual focus ring.
**Action:** Use the 'visually hidden' CSS pattern (`opacity: 0; width: 0; height: 0; position: absolute;`) instead of `display: none` for the hidden `<input>`. Add a `:focus-visible` outline to adjacent elements (like the slider) to ensure visibility, and always provide `role="switch"` and an `aria-label`.
