## 2026-03-18 - Accessible Hidden Inputs and Playwright
**Learning:** `display: none` completely removes an element from the accessibility tree, breaking screen reader and keyboard accessibility for custom inputs (like toggle switches). In Playwright, `page.wait_for_selector` fails on visually hidden elements (`opacity: 0`), requiring testing strategies to target adjacent visible elements or use explicit timeouts. Actual keyboard events (`page.keyboard.press("Tab")`) are needed to trigger visual `:focus-visible` rings, rather than programmatic `.focus()`.
**Action:** Always use the 'visually hidden' CSS pattern (`opacity: 0`, `position: absolute`, `width: 0`, `height: 0`) with a `:focus-visible` outline on adjacent visible elements for custom inputs. When writing Playwright tests for these inputs, rely on keyboard simulation and adjacent element visibility.
