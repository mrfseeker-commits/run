## 2024-05-24 - Accessible Custom Checkbox Toggle Switch
**Learning:** Using `display: none` on custom toggle switch inputs completely removes them from the accessibility tree and prevents keyboard navigation. Also, Playwright scripts struggle to find elements via `page.wait_for_selector` when using accessible hidden patterns (`opacity: 0`). When applying `role="switch"` to native `<input type="checkbox">` elements, the `aria-checked` attribute is implicitly handled by the browser based on the checked state.
**Action:** Always use visually hidden patterns (`opacity: 0`, `position: absolute`, `width: 0`, `height: 0`) instead of `display: none` for accessible hidden inputs. Ensure you add `:focus-visible` to adjacent visible elements (like `.slider`) to provide clear visual focus indicators for keyboard navigation. Include `role="switch"` and localized `aria-label`s for screen reader support. When verifying with Playwright, rely on other wait conditions or target adjacent visible elements and use actual keyboard event simulation (e.g., `page.keyboard.press("Tab")`) to test the `:focus-visible` state.
